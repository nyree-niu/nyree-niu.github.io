Building a website requires three core languages: HTML, CSS and JavaScript.

HTML determine the content and structure, CSS controls the style and appearance, and JavaScript makes the website fun and interesting, by adding interactivity and dynamic behaviors.

Liking creating a cartoon character, HTML determines blueprint such as how big the eyes are and where they are placed, CSS determines the color and shape of eyes, and JavaScript make the cartoon character interact with audience: if you blink, the cartoon blinks backs.

In addition to the three core languages, there is another tool which is essential for website developers. The Document Object Model (DOM) is a programming interface that web browsers create from HTML. It represents a webpage as a tree of objects. Each part of the page—like headings, paragraphs, buttons, or images—becomes a node in this tree. Web developers use JavaScript to manipulate the DOM—they can add, remove, or change elements on a webpage dynamically without reloading the page.

In other words, DOM is a live model of your webpage that JavaScript can talk to and change in real time. Google Chrome is one of the most popular browsers that lets you inspect and manipulate the DOM. User can access to it by opening any website of interest, and use short cut Command + Option + J (Mac).

Control flow is the order in which steps in a program are executed. Just like in real life, you often follow a sequence of steps to complete a task. For example, I want to make an espresso to start my day:

  1. Turn on the coffee machine.
  2. Measure coffee beans.
  3. Pour beans into the grinder.
  4. Attach the portafilter to the grinder.
  5. Grind the beans.
  6. Move the portafilter to the espresso machine.
  7. Place an empty cup under the portafilter.
  8. Start brewing.
  9. If water is low, refill and restart brewing.
  10. Pour milk into a steaming cup.
  11. Steam the milk to the desired temperature.
  12. Stop steaming.
  13. Pour milk into the brewed espresso.
At step 9, there’s a small loop: if the machine needs more water, refill it and loop back to step 8 to restart brewing. Once brewing begins, the process continues normally. The loop stops when the conditions meet (water refilled), and the loop won't start if there are enough water (refill not need).

  • It can be write once and be used many times.
  • It can also take input (called parameters) and return output (results).
For example,

Return " Hello " + name
}
